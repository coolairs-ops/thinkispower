# 交付页功能完善 — 规划

> 2026-06-01 | 在线展示、在线部署、源码下载三大功能

---

## 一、当前问题

```
API 返回:                   前端显示:
  status: "completed"  →   "已交付 — 代码已生成，可下载源码或在线访问"
  productionUrl: null  →   无在线部署链接
  latestBuild: null    →   无下载链接
  generatedFiles: []   →   无文件列表
```

**根因**: 项目被假标记为 `completed`（无实际交付数据），或数据存在但 URL 为 null。

---

## 二、目标状态

交付页完成态应有三块功能卡片：

```
┌─────────────────────────────────────────────┐
│ ✅ 已交付 — 4 个文件已生成                    │
└─────────────────────────────────────────────┘

┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│ 🌐 在线访问   │ │ 📦 下载源码   │ │ 📂 文件清单   │
│              │ │              │ │              │
│ 可访问URL     │ │ sourceZipUrl │ │ 8 个文件     │
│ [打开应用]    │ │ [下载]       │ │ backend/...  │
│ [复制链接]    │ │              │ │ frontend/... │
└──────────────┘ └──────────────┘ └──────────────┘
```

---

## 三、解决方案

### 3.1 后端修复

| 文件 | 改动 | 说明 |
|------|------|------|
| `delivery-evaluation.service.ts` | 交付完成后确保 productionUrl 不为 null | 已在 Phase C 中实现 DeployPipeline |
| `delivery.service.ts` | generatedFiles 返回空时尝试从文件系统读取 | 已有 walkDir 逻辑，需确保容器文件持久化 |

### 3.2 前端修复 (本次重点)

| 改动 | 说明 |
|------|------|
| 已交付但无数据 → 提示"交付数据异常，请重新交付" | 不显示误导性的成功消息 |
| 已交付有数据 → 三栏卡片 | 在线访问 + 下载源码 + 文件清单 |
| 在线访问 → 显示 URL + 打开按钮 + 复制按钮 | 已有逻辑，需确保 URL 存在 |
| 下载源码 → 显示文件名 + 下载按钮 | 使用 sourceZipUrl |
| 文件清单 → 分类展示 | 前端/后端/数据库/配置 分组 |

### 3.3 数据异常处理

当 `status === 'completed'` 但 `!latestBuild && !productionUrl` 时：

```
┌─────────────────────────────────────────────┐
│ ⚠️ 交付数据异常                              │
│                                             │
│ 项目状态显示为"已交付"，但未找到交付产物。    │
│ 这可能是早期版本的遗留数据。                  │
│                                             │
│ [重新交付]                                   │
└─────────────────────────────────────────────┘
```

## 五、验证标准

1. 有 delivery 数据的项目 → 三栏卡片全部显示且可点击
2. 无 delivery 数据的 completed 项目 → 显示异常提示 + 重新交付按钮
3. 下载链接点击 → 浏览器下载 tar.gz
4. 在线访问链接 → 打开 MinIO 部署页
5. 文件清单 → 显示完整文件列表
