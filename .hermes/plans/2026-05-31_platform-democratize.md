# Think-is-power 全民化优化方案

> 2026-05-31 | 从"服务PM"到"服务所有人"

## 核心思路

不动生成引擎，只改**输入层**。让不同能力的用户用不同的方式描述需求，平台通过对话把信息问到"能生成方案"为止。

```
现有架构:
  描述框 → Plan → Demo → 自迭代 → 交付

优化后:
  对话引导 → 信息完备度检测 → Plan → Demo → 自迭代 → 标准化交付 + 报价
  ↑ 新增层     ↑ 新增层                                    ↑ 已规划(P0+P2)
```

## 改动清单

### 第一层：智能对话引导（新增）

**位置**: `apps/web/src/app/projects/[id]/page.tsx` — 当前项目详情页

**改为**: 对话式引导模式。用户创建项目后不再是白底文本框，而是：

```
┌─────────────────────────────────────┐
│ 🤖 你想做什么？                      │
│                                     │
│ [文本框]                             │
│ [发送]                               │
│                                     │
│ 💡 快速开始:                         │
│ [记账工具] [管理后台] [小程序] [其他]  │
└─────────────────────────────────────┘
```

用户说一句话 → 平台分析需求缺口 → 逐个问：

```
平台: 是一个不错的想法。我先确认几个问题：

平台: ① 谁会用这个工具？ [仅我自己] [小团队] [对外服务]
      ② 数据多吗？（条数） [<1000] [1000-10万] [>10万]
      ③ 需要哪些功能？ [分类统计] [拍照/上传] [分享] [搜索] [提醒]

用户: [点选完成]

平台: 好的，还需要明确一下：
      → "分类统计"需要按什么分类？ [类别] [时间] [金额范围]
      → "分享"是分享给谁？ [微信链接] [导出Excel] [邀请协作者]
```

**关键设计**: 平台问问题的顺序由**信息完备度引擎**决定——只有当前回答触发了信息缺口时才追问。

**后端改动**: `apps/api/src/services/discovery-engine.service.ts`（新建）
- 输入: 用户消息 + 当前已收集的信息
- 输出: `{ nextQuestion, missingInfo: [...], completeness: 35% }`
- 调 DeepSeek 决定下一轮问什么
- 满 70% 自动建议"信息足够了，要生成方案吗？"

### 第二层：信息完备度引擎（新增）

**位置**: `apps/api/src/services/completeness-checker.service.ts`（新建）

每次用户回答后，评估当前信息覆盖度：

```typescript
interface CompletenessReport {
  overall: number;              // 0-100
  productForm: 'known' | 'missing';     // 产品形态（网页/App/小程序）
  targetUsers: 'known' | 'missing';     // 目标用户
  coreFeatures: 'known' | 'partial';    // 核心功能
  dataModel: 'known' | 'partial';       // 数据模型
  businessRules: 'known' | 'missing';   // 业务规则
  scale: 'known' | 'missing';           // 规模预估
  gaps: string[];                // 缺失项说明
}
```

`overall >= 70` → 自动提示"可以生成方案了"
`overall < 40` → 继续追问
`overall >= 90` → 直接生成（产品经理路径）

### 第三层：缺口可视化 + 成本预判（修改现有页面）

**位置**: `apps/web/src/app/projects/[id]/page.tsx` — 项目详情页右侧

新增侧边栏：

```
┌─────────────────────────┐
│ 📊 方案完备度  65%      │
│                         │
│ ✅ 产品形态  网页        │
│ ✅ 目标用户  个人用户    │
│ ⚠️  数据模型  部分       │ ← 点开能看到"还缺：字段类型、关联关系"
│ ❌ 业务规则  未填写      │
│                         │
│ 💰 预估成本             │
│    自动完成: 80%        │
│    需人工: ¥300-600     │
│    [查看明细]            │
│                         │
│ [继续完善需求]           │
│ [生成方案 →]            │ ← 完备度>=70才可点
└─────────────────────────┘
```

### 第四层：模板化输入辅助（新增组件）

**位置**: `apps/web/src/components/` — 可复用组件

对于"数据字典"这类需要结构化输入的，不要求用户自己填表，而是：

```
平台: 你提到需要"分类统计"。统计的数据主要是？
      [金额] [时间] [类别] [自定义]
      
用户: 金额和时间

平台: （后台自动构建）
      → 数据对象: 交易记录
      → 字段: 金额(decimal), 时间(datetime), 类别(string)
      
      请确认或修改：
      ┌──────────────────────────┐
      │ 交易记录                  │
      │ ├ 金额    decimal   ✅    │
      │ ├ 时间    datetime  ✅    │
      │ ├ 类别    string    ✅    │
      │ └ + 添加字段             │
      └──────────────────────────┘
```

PM 觉得烦可以跳过手动确认，点"全自动"让平台自己定。

### 第五层：不同入口，不同起点（前端路由）

**位置**: 首页 `apps/web/src/app/page.tsx`

当前首页只有登录框。改为四个入口：

```
┌─────────────────────────────────────────┐
│  Think-is-power                          │
│  把你的想法变成软件                        │
│                                         │
│  [快速开始]  → 一句话，对话引导           │
│  [我有想法]  → 自由描述，平台提问         │
│  [我是专业用户] → PRD/数据字典完整表单    │
│  [看看Demo]  → 浏览已有项目样例          │
└─────────────────────────────────────────┘
```

- **快速开始**: 纯对话模式，0门槛
- **我有想法**: 对话 + 快速模板选择
- **专业用户**: 直接到完整表单（数据字典 + 权限 + 部署配置）

## 实现优先级

| 优先级 | 内容 | 改动量 | 理由 |
|--------|------|--------|------|
| **P0** | 第二层：完备度引擎 | 1新建文件 | 没有它，对话不知道问什么 |
| **P0** | 第四层：模板化输入 | 1-2个组件 | PM也能用，普通人也能用 |
| **P1** | 第一层：对话引导 UI | 1改（项目详情页） | 依赖 P0 的完备度数据 |
| **P2** | 第五层：首页入口 | 1改（首页） | 视觉改版，不依赖后端 |
| **P2** | 第三层：缺口侧边栏 | 1改（项目详情页） | 增强体验但非核心 |

## 与现有功能的衔接

```
现有功能                   优化后
    
创建项目                   → 保留，但创建后自动进入对话引导
需求描述框                 → 保留，但新增对话气泡 + 选项按钮
Plan生成                   → 保留，但触发时机改为"完备度>=70自动提示"
Demo生成                   → 保留，不变
自迭代                     → 保留，不变
交付                       → 保留，但产物注入 P0 企业模板
```

## P0 落地步骤

**Step 1**: 新建 `completeness-checker.service.ts`
```
输入: messages[], 当前已选选项
输出: CompletenessReport
逻辑: 
  - 有 productForm? → +20
  - 有 targetUsers? → +15
  - 有 coreFeatures(≥2)? → +20
  - 有 dataModel(≥1表)? → +15
  - 有 businessRules? → +10
  - 有 scale? → +10
  - 有 acceptanceCriteria? → +10
```

**Step 2**: 新建 `discovery-engine.service.ts`
```
输入: CompletenessReport + 最后一条用户消息
输出: nextQuestion (字符串) 或 "完备度已达标"
逻辑: 调 DeepSeek，让它根据缺口生成合适的下一个问题
```

**Step 3**: 新建 `suggestion-picker` 组件
```
展示给用户: 问题 + 2-4个选项按钮 + 自定义输入
用户点选 → 自动发送消息 → 更新完备度
```

**Step 4**: 修改项目详情页，底部改为对话流
