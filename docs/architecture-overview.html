⚙️ 一句话做软件平台 2.0 · 顶层架构设计图

面向普通用户，平台只呈现"说需求、看方案、看预览、点选修改、获取交付"；面向内部系统，新增"事件驱动管道 + 经验资产层"，批注意见通过 事件管道 自动流转：API 存储 → OpenClaw AI 决策拆任务 → PipelineService 工作流执行 → Cloudecode 改代码 → Demo 更新。

🎯 用户看到的核心承诺

1. 说人话就行用户只描述想做什么，平台负责追问和整理。
2. 先看样子再确认先生成可点击预览，降低做错方向的风险。
3. 哪里不对点哪里用户直接在页面上批注，不需要懂技术。
4. 多形态交付可在线使用，也预留源码、项目包、仓库交付等高级服务。

🏰 平台内部护城河

事件驱动管道API 只存数据发事件，不关心谁处理；各层独立演进。
AI 决策（总工）OpenClaw 分析意见、拆解任务、决定修改方案。
经验资产沉淀决策日志、错误模式、项目复盘持续积累，越用越聪明。
失败自动恢复Pipeline 重试、错误匹配库、回滚机制保障稳定。

🛡️ 横切保障层 · 用户感知不到，但平台始终执行

📍 点选定位用户点页面哪里，平台通过 data-module-key + postMessage 定位修改位置。
💓 运行检查持续检查任务、页面、接口、测试、部署是否正常。
🧯 出错恢复失败时自动重试、匹配错误模式修复，必要时回到上一个可用版本。
🔒 项目隔离每个用户、每个项目都有独立空间，互不影响。
🧼 信息过滤用户界面只出现普通人能理解的文案，不暴露内部术语。

🟢 方案确认回路
平台整理方案 → 用户确认 → 再生成预览或进入开发。
🟡 批注修改管道
提交意见 → feedback.created → OpenClaw 拆 Task → tasks.created → Pipeline 逐任务执行 → Cloudecode 改代码 → tasks.completed → Demo 刷新。
🔵 自动测试回路
开发完成 → 自动检查 → 发现问题 → 自动修复 → 再检查。
🔴 出错恢复回路
Task 失败 → 自动重试 → 匹配错误模式 → 修复或回滚。
🟣 多形态交付回路
在线使用 → 用户验收 → 按服务等级提供源码或项目包。

🏰 经验资产闭环：每做一个项目，平台都变得更聪明

1. 记录判断
每次 AI 拆任务、执行结果、失败原因都写入 DecisionLog。
2. 记录错误
沉淀错误特征、触发场景、推荐修复方式和修复成功率。
3. 自动复盘
项目完成后总结需求、修改次数、主要问题、交付方式和可复用经验。
4. 反哺下一次
类似项目自动推荐模板、关键追问、测试项和处理策略。
🔵 用户使用层：聊天、确认方案、看预览、点选批注、获取交付
🟠 项目管家层：整理需求、拆任务、生成方案、通知进度
🟣 AI 决策层：事件驱动、意见分析、任务拆解、验收标准
🩷 工作流执行层：PipelineService、逐 Task 执行、重试、决策日志
🟢 执行交付层：代码生成、批注定位、质量检查、多形态交付
设计原则：用户侧保持极简，只看到"说需求、看方案、看预览、点选修改、获取交付"；平台内部新增事件驱动管道（API → OpenClaw AI 决策 → PipelineService 执行 → Cloudecode 改代码），所有通信通过 @nestjs/event-emitter 事件解耦。各层可独立演进：Phase 1 使用进程内事件，后续可替换为 n8n 工作流或消息队列。所有内部方法论、工具名、执行代号都不得出现在用户可见界面、通知、错误提示和帮助文档中。
