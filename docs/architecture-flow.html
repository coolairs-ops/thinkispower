👤 用户

在 Demo 页面批注模式下点击元素 → 填写意见 → 提交
  1. 点击 Demo 页面中的元素 → 侧边栏出现表单
  2. 填写修改意见："表格增加一列操作按钮"
  3. 点击"提交意见"
POST /feedback

存储层 API · Think-is-power

NestJS 网关 + 数据持久化
  1. 存入 FeedbackItem 到 PostgreSQL
  2. 更新 Project.status = 'awaiting_demo_feedback'
  3. 发布事件：📢 feedback.created
{ projectId: "uuid", feedbackId: "uuid" }
📢 feedback.created

决策层 · 总工 OpenClaw

AI 分析批注意见 → 拆解为可执行的任务清单
  1. 收到 📢 feedback.created
  2. 读取 FeedbackItem.comment + 当前 Project.demoHtml
  3. AI 分析：改哪里？怎么改？需要拆成几个任务？
  4. 产出 1～N 个 Task，写入 DB
  5. 发布事件：📢 tasks.created
┌──────────────────────────────────────┐ │ type: "frontend" │ │ title: "列表页表格增加操作按钮列" │ │ moduleKey: "list" │ │ description: "在表格 thead 加'操作' │ │ 列，每行末尾加编辑和删除按钮..." │ │ acceptanceCriteria: [ │ │ "thead 有'操作'列", │ │ "按钮有 data-module-key 属性" │ │ ] │ └──────────────────────────────────────┘
📢 tasks.created

工作流层 · 项目经理 n8n

可靠的工作流执行引擎 — 重试、超时、并行
  1. 收到 📢 tasks.created
  2. 读取所有 status = 'pending' 的 Task
  3. 对每个 Task 循环： 更新 status='running' → 调 Cloudecode → 成功 → completed / 失败 → 重试(最多3次) → → failed
  4. 为每个 Task 记录 DecisionLog
  5. 所有 Task 完成后，回调 API
  6. 发布事件：📢 tasks.completed

执行层 · 开发 Cloudecode

接收具体任务 → 修改代码 → 报告结果
  1. n8n 调用 POST /cloudecode/execute
  2. 读取当前 demoHtml
  3. 调用 DeepSeek 生成修改后的 HTML
  4. 验证 HTML 结构正确
  5. 返回：{ success, html, changedFiles, testReport }

回调 API 收到 n8n 回调

更新数据库 → 用户看到新 Demo
  1. 更新 FeedbackItem.status = 'resolved'
  2. 更新 Project.demoHtml = 新 HTML
  3. 更新 Project.status = 'demo_ready'
  4. 用户刷新页面 → 看到新版 Demo
