① 需求澄清 Discovery Module

通过AI多轮自然对话探索用户真实需求。系统扮演产品经理角色，每轮只问一个关键问题， 覆盖 8 个维度（目标用户、痛点、场景、核心价值、产品形态、MVP范围、成功标准、参考约束）。 ≥7 维度完备或对话 ≥12 轮后自动输出结构化 PRD。

⑤ 自迭代质量评估 L1/L2/L3 Sensors + Sensor Fusion

核心机制说明

🔁 批注反馈闭环

用户点选 Demo 中任意带 data-module-key 的元素 → 弹出批注框 → AI 自动判定 Bug/变更 → 拆解任务 → Pipeline 执行 → Demo 自动更新 → 可继续批注。每次修改前自动快照，支持回滚。

🧬 三级传感器融合

L1 静态（HTML结构+语法+质量门禁+标注覆盖率）→ L2 运行时（截图像素对比+健康度）→ L3 语义（Qwen交叉验证+需求追溯+反馈闭环）。SensorFusion 加权综合评分。

🔧 多引擎生成链路

Demo 生成优先级：DeepSeek 直调（快）→ Cloudecode → N8N 工作流。交付导出：CC Bridge 流水线 → Cloudecode → N8N。失败自动降级，不影响主流程。

📋 规格冻结机制

规格冻结后创建的新反馈自动判定是 Bug（在验收场景/核心功能内）还是变更请求（在 outOfScope 或含"新增"关键词），防止范围蔓延。

🚦 全局迭代锁

自迭代使用数据库全局锁 SystemLock 保证同一时间只有一个项目在迭代。30 分钟超时自动清除陈旧锁，避免死锁。

📊 案例复盘 & 经验

交付完成后自动生成 AI 复盘报告（主要错误、修复策略、可复用经验），并跨项目沉淀经验推荐（问题模板、测试用例、修复策略、风险预警）。
